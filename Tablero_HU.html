Estados de Historias de Usuario – Enfoque Célula

Resumen operativo para control real del avance, alineado a Azure DevOps y al trabajo diario de la célula.

Resumen de Estados (visión célula)

  • Nueva / Aprobada: La célula no puede trabajar. Pendiente de negocio.
  • Refinada: La célula entiende, acepta y se compromete.
  • Desarrollo: DEV trabajando activamente.
  • Fin Desarrollo: DEV terminó y entrega a QA con correo de liberación.
  • Pruebas QA: QA valida. DEV solo soporta.
  • Certificado QA: Lista para negocio.
  • Certificado UAT (Review): Validación y aprobación del negocio.
  • Producción: Despliegue y estabilización.
  • Cerrada: HU estable, capacidad liberada.
Estado ¿Quién mueve el estado? Qué significa realmente Evidencia o Artefacto
Nueva / Aprobada BA HU creada, sin validar HU Escrita en Azure
Refinada BA Negocio aprobó alcance, Entendida y lista para trabajar Nota en HU con Link de Grabación de Teams
Desarrollo DEV Código en construcción

Cerrada Scrum Master HU finalizada

Flujo Visual de Estados

Caminos del Flujo

Camino Feliz

  • Nueva → Aprobada → Refinada
  • Refinada → Desarrollo → Fin Desarrollo
  • Fin Desarrollo → Pruebas QA → Certificado QA
  • Certificado QA → Pruebas / Certificado UAT (Review) → Producción
  • Producción → Cerrada

Caminos No Felices

  • Reproceso técnico: Pruebas QA → Desarrollo
  • Observaciones de negocio: UAT → BA valida contra Criterios de Aceptación y define BUG o Evolutivo
  • Observaciones de negocio: Si esta dentro de Criterios de Aceptación BUG → Desarrollo
  • Observaciones de negocio: Si esta Fuera de Alcance → Nuevas HU
  • Bloqueo: Cualquier estado → Bloqueada
  • Cancelación: Cualquier estado → Cancelada

Estados Especiales

  • Bloqueada: Impedimento real. El Scrum Master debe gestionar desbloqueo.
  • Cancelada: Ya no aplica. Se libera capacidad inmediatamente.

Regla de Oro

Si la célula no puede avanzar, el estado está mal.

Los estados reflejan realidad operativa, no intención ni presión.

La actualización de notas (@notas) es obligatoria para bloqueos, dudas y aclaraciones.

La ausencia de notas puede generar reprocesos y retrasos.
